# URL Shortener API

A simple URL shortener REST API built with Kotlin and Spring Boot.

## Prerequisites

- Java 21+

## Running Locally

```bash
./gradlew bootRun
```

The application starts on `http://localhost:8080`.

## Running Tests

```bash
./gradlew test
```

## API Documentation

Swagger UI is available at `http://localhost:8080/swagger-ui/index.html` when the application is running locally. It provides an interactive interface to explore and test all API endpoints.

The raw OpenAPI spec is available at `http://localhost:8080/v3/api-docs`.

## Design

### Shorten a URL
Accepts a `POST` request with the original URL in the request body. The URL is validated to ensure it is well-formed. A unique slug is generated by base62-encoding a cryptographically random `Long` value, guaranteeing short, unique codes without a database. The slug and the constructed short URL are returned in the response.

### Redirect to Original URL
Accepts a `GET` request with the slug as either a path variable (`/redirect/{slug}`) or a query parameter (`/redirect?slug=`). Looks up the original URL in an in-memory map and returns a `302 Found` with a `Location` header pointing to it. Returns `404` if the slug is unknown.

### Get URL Info
Accepts a `GET` request with the slug as either a path variable (`/shorten/{slug}`) or a query parameter (`/shorten?slug=`). Returns the shortened URL, original URL, and the timestamp at which the URL was shortened. Returns `404` if the slug is unknown.

### Slug Generation
Slugs are always 7 characters long, generated by base62-encoding (`0-9`, `A-Z`, `a-z`) a cryptographically random `Long` value produced by `SecureRandom`. The value is bounded to `62^7 ≈ 3.5 trillion` possible values. Collisions are astronomically unlikely and handled by retrying with a new random value.

### Rate Limiting
No rate limiting is implemented. For a production service, rate limiting should be applied to `POST /api/v1/shorten` to prevent abuse. This is typically handled at the API gateway level (e.g. Spring Cloud Gateway) or via an in-process library such as Bucket4j.

### Duplicate URLs
The `POST /api/v1/shorten` endpoint is idempotent — if the same long URL is shortened more than once, the existing slug is returned rather than generating a new one. This is implemented via a reverse lookup map (`originalUrl → slug`) that is checked before generating a new slug.

### Storage
URLs are stored in a `ConcurrentHashMap` keyed by slug, making the implementation thread-safe without requiring a database. Note that the map grows indefinitely for the lifetime of the application — no eviction or TTL policy is implemented. For a production system, a persistent store or a cache with eviction (e.g. Redis) would be more appropriate.

## API Endpoints

### Shorten a URL
```
POST /api/v1/shorten
Content-Type: application/json

{ "url": "https://www.originenergy.com.au/electricity-gas/plans.html" }
```
**Response:**
```json
{ "slug": "a1B2c3D", "url": "http://short.ly/a1B2c3D" }
```

### Redirect to Original URL
```
GET /api/v1/redirect/{slug}
GET /api/v1/redirect?slug={slug}
```
Returns HTTP `302 Found` with a `Location` header pointing to the original URL.

### Get URL Info
```
GET /api/v1/shorten/{slug}
GET /api/v1/shorten?slug={slug}
```
**Response:**
```json
{
  "shortenedUrl": "http://short.ly/a1B2c3D",
  "originalUrl": "https://www.originenergy.com.au/electricity-gas/plans.html",
  "createdAt": "2026-03-24T10:00:00Z"
}
```

## curl Examples

### Shorten a URL
```bash
curl -v http://localhost:8080/api/v1/shorten \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.originenergy.com.au/electricity-gas/plans.html"}'
```
```
> POST /api/v1/shorten HTTP/1.1
> Host: localhost:8080
> Content-Type: application/json
> Content-Length: 71
>
< HTTP/1.1 200
< Content-Type: application/json
<
{"slug":"dDsWqfe","url":"http://short.ly/dDsWqfe"}
```

### Redirect via path variable
```bash
curl -v http://localhost:8080/api/v1/redirect/dDsWqfe
```
```
> GET /api/v1/redirect/dDsWqfe HTTP/1.1
> Host: localhost:8080
>
< HTTP/1.1 302
< Location: https://www.originenergy.com.au/electricity-gas/plans.html
< Content-Length: 0
```

## Error Responses

| Status | Reason |
|--------|--------|
| `400 Bad Request` | Invalid or missing URL / blank slug |
| `404 Not Found` | Slug does not exist |
